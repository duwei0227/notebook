Java单元测试

单元测试：只测试系统中最原子化的行为单元的自动化测试–测试核心逻辑

简单设计4原则：

测试变绿–测试通过
没有重复–代码没有重复
揭示意图–测试什么功能
最少元素

单元测试策略：变红(单元测试)->变绿(烂代码)->重构(优化)

测试替身

SUT System Under Test
DOC Dependence On Component

Mockito

  • Mock – 不关心数据的输入和输出，验证mock的方法是否被调用过
    • 验证行为
    • 屏蔽行为
  • Stub – 数据的伪造，替换真实数据
    • 状态模拟
  • Spy
    • 假中有真
